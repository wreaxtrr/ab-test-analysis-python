# A/B-тест новой посадочной страницы e-commerce

## Описание проекта

В этом проекте проведён анализ A/B-теста новой посадочной страницы e-commerce сайта.

Часть пользователей попала в контрольную группу и видела старую версию страницы, а часть пользователей попала в тестовую группу и видела новую версию страницы.

Цель проекта — проверить, улучшила ли новая страница ключевые продуктовые и денежные метрики:

- conversion rate;
- ARPU;
- average check.

По итогам анализа необходимо принять продуктовое решение: стоит ли выкатывать новую страницу на всех пользователей.

## Используемые данные

В проекте используется учебный датасет с Kaggle: **A/B Testing**.

Датасет содержит информацию о пользователях e-commerce сайта, участвовавших в A/B-тесте новой посадочной страницы.

Основные поля датасета:

- `user_id` — идентификатор пользователя;
- `timestamp` — время события;
- `group` — экспериментальная группа пользователя;
- `landing_page` — версия посадочной страницы;
- `converted` — факт совершения покупки;
- `purchase_amount` — сумма покупки;
- `age` — возраст пользователя;
- `gender` — пол пользователя;
- `location` — страна пользователя;
- `session_duration` — длительность сессии;
- `pages_visited` — количество просмотренных страниц;
- `device_type` — тип устройства.

Датасет не включён в репозиторий. Для воспроизведения анализа нужно скачать CSV-файл с Kaggle и положить его рядом с ноутбуком под названием:

```text
AB Testing Data.csv
```

## Этапы анализа

В рамках проекта были выполнены следующие шаги:

1. Загрузка и первичный осмотр данных.
2. Проверка качества данных:
   - пропуски;
   - дубликаты;
   - повторяющиеся пользователи;
   - пользователи, попавшие в несколько групп;
   - соответствие между `group` и `landing_page`.
3. Подготовка признаков для анализа.
4. Исследовательский анализ данных.
5. Расчёт ключевых метрик:
   - conversion rate;
   - ARPU;
   - average check.
6. Расчёт uplift и абсолютных различий между группами.
7. Статистическая проверка гипотез:
   - z-test для conversion rate;
   - t-test для ARPU;
   - t-test для average check.
8. Расчёт доверительных интервалов.
9. Формирование итоговой бизнес-рекомендации.

## Основные результаты

По итогам анализа были получены следующие результаты:

| Метрика | Control | Treatment | Результат |
|---|---:|---:|---|
| Conversion rate | 11.87% | 17.95% | статистически значимый рост |
| ARPU | 4.45 | 6.76 | статистически значимый рост |
| Average check | 37.50 | 37.68 | статистически значимого изменения нет |

Новая посадочная страница показала заметный рост conversion rate и ARPU. При этом средний чек практически не изменился.

Это означает, что основной эффект новой страницы заключается не в увеличении размера покупки, а в росте вероятности совершения покупки.

## Итоговая рекомендация

Новую посадочную страницу можно рекомендовать к внедрению, так как она статистически значимо увеличила:

- conversion rate;
- ARPU.

При этом статистически значимого изменения average check не обнаружено.

После внедрения рекомендуется дополнительно отслеживать стабильность эффекта на всей пользовательской базе: conversion rate, ARPU, average check, а также результаты по отдельным сегментам пользователей.

## Ограничения анализа

У анализа есть несколько ограничений:

- датасет является учебным;
- период наблюдений слишком длинный для типичного A/B-теста;
- не учитывались возможные внешние факторы: сезонность, акции, маркетинговые кампании;
- основной анализ проведён на общем уровне без подробной сегментации;
- для денежных метрик использовался t-test как простой и интерпретируемый метод сравнения средних значений.
